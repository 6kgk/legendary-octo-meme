# 【职教通】职业图谱映射与实习模块数据模型设计

根据林老板和项目经理的要求，我完成了“3+证书”考纲与深圳产业岗位的映射研究，并初步设计了实习模块的底层数据模型。

## 1. 职业图谱映射逻辑 (The Mapping Engine)

我们将学生的文化课学习表现（数学、英语）通过 AI 转化为可衡量的**职业通用能力**，并精准对接深圳本地岗位的技能需求。

### 1.1 数学 (Mathematics) -> 逻辑与数据能力
| 考纲知识点 | 映射职业能力 | 深圳典型实习岗位 |
| :--- | :--- | :--- |
| **集合与逻辑** | 条件判断、严谨性 | 软件黑盒测试、硬件质检 (IQC/FQC) |
| **函数与方程** | 趋势分析、参数调节 | 智能制造设备调机、初级会计助理 |
| **数列与计算** | 模式识别、计算精度 | 仓库库存管理、SMT 贴片机维护 |
| **概率与统计** | 数据敏感度、风险预估 | 跨境电商运营助理 (销量预测) |

### 1.2 英语 (English) -> 沟通与国际化能力
| 考纲知识点 | 映射职业能力 | 深圳典型实习岗位 |
| :--- | :--- | :--- |
| **基础词汇/语法** | 文档理解、基础沟通 | 进出口报关助理、外贸客服 |
| **阅读理解** | 信息提取、逻辑串联 | 电子元器件英文手册查阅、海外仓维护 |
| **应用写作** | 商务规范、书面表达 | 跨境电商产品描述编写、英文邮件回复 |

## 2. 实习模块数据模型设计 (Data Schema)

为了实现“能力分自动生成简历”和“岗位精准匹配”，我设计了以下核心数据结构。

### 2.1 学生能力雷达表 (`Student_Skill_Radar`)
存储学生在应用内通过刷题和 AI 辅导累积的能力值。
```json
{
  "student_id": "UUID",
  "radar_data": {
    "logic_score": 85,        // 由数学(集合/函数)计算得出
    "data_analysis": 70,      // 由数学(概率/统计)计算得出
    "english_literacy": 78,   // 由英语阅读/写作计算得出
    "patience_index": 92,     // 由错题复盘率、学习时长计算得出
    "technical_cert": ["计算机一级", "电工初级"] // 录入的专业证书
  },
  "last_updated": "Timestamp"
}
```

### 2.2 企业岗位需求表 (`Internship_Position`)
存储深圳合作企业发布的岗位及“门槛分数”。
```json
{
  "position_id": "UUID",
  "company_name": "嘉立创 / 大疆 / 跨境大卖家",
  "job_title": "硬件测试实习生",
  "requirement_threshold": {
    "logic_score": 80,
    "patience_index": 85
  },
  "job_tags": ["严谨", "电子基础", "深圳龙华"],
  "ai_resume_template": "Technical_Template_V1"
}
```

### 2.3 AI 简历生成逻辑 (`Resume_Generator`)
系统根据 `Student_Skill_Radar` 的高分项，自动调用 AI 润色简历：
*   **示例**：如果学生“逻辑分”高，简历中自动加入：“具备极强的逻辑推导能力，在数万名学生中逻辑排名前 10%，擅长精准定位问题原因。”

## 3. 技术实施建议

1.  **动态权重算法**：建立一个简单的加权模型。例如：针对“跨境电商”岗位，英语的权重占 60%，数学统计占 40%。
2.  **简历“整容”计划**：利用大模型将学生的“考试分数”翻译成企业 HR 看得懂的“职场语言”。
3.  **隐私保护**：学生的考试具体错题不向企业开放，仅开放经过脱敏的能力维度分。

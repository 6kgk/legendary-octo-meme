# “职教通”Token 降本增效：结构化 Prompt 与本地逻辑优化方案

## 一、 “Token 瘦身”核心策略 (Cost Control Strategy)
我们的目标是：**简单的活本地干，难的活精准点，长的话分段说。**

### 1. 结构化系统指令 (Structured System Prompts)
将冗长的角色描述简化为 Markdown 格式的指令集，减少大模型在“理解身份”上的 Token 消耗。

**[优化前示例]**：你是一个非常专业的中职教育专家，你需要用大白话给学生讲题，不要直接给答案，要引导他们，要注意他们的自尊心...（约 200 Tokens）
**[优化后 (System Prompt)]**：
```markdown
# Role: 广东职教专家
# Goal: 启发式辅导，严禁直给答案。
# Style: 粤语口语化/大白话，降维讲解。
# Logic: 1.确认基础 -> 2.分步引导 -> 3.反馈职业点。
```
*（节省约 70% 的系统指令消耗）*

## 二、 “分级引导”交互协议 (Multi-stage Interaction)
不再一次性生成冗长的完整解析，采用“按需索取”模式。

| 交互阶段 | 长度限制 | AI 任务指令 | 预期 Token |
| :--- | :--- | :--- | :--- |
| **第一步：微型提示** | < 50 Tokens | 识别题型，给出一个关键公式或第一个思考切入点。 | ~40 |
| **第二步：关键拆解** | < 150 Tokens | 只有学生反馈“不懂”或“算了一半”时，再解释核心逻辑。 | ~120 |
| **第三步：完整复盘** | < 200 Tokens | 确认学生做对后，总结知识点并映射职业能力。 | ~180 |

## 三、 本地逻辑下放：【避雷审计】预处理词库
在调用 AI 审计合同前，先由后端代码执行关键词匹配，拦截 60% 的常见风险。

**[建议邓绍鹏实现的正则/关键词逻辑]**：
*   **违规收费**：`押金|保证金|服装费|管理费|培训费|入职费`
*   **非法扣证**：`扣押|保管|身份证|毕业证|证件原件`
*   **异常工时**：`两班倒|12小时|夜班|无休|自愿加班`
*   **非法薪资**：`无薪|试用期零薪|不发工资|管理费抵扣`

**[逻辑流]**：
1. 本地扫描命中的风险项 $\rightarrow$ 直接在前端显示 **[已通过本地风险库识别]**。
2. 剩余复杂的法律表述 $\rightarrow$ 发送给 AI 进行 **[深度合规性分析]**（仅发送相关段落，不发送全篇）。

## 四、 简历背书：【模版化】输出引擎
减少 AI 自由发挥的字数，采用“填空式”生成。

**[AI 任务指令优化]**：
输入：学生数学几何正确率 95%，速度优。
指令：从字典中选出最匹配的【标签】和【话术】，格式：[标签]: [话术]。
**[预期输出]**：
[精密空间感]: 具备极佳的三维构图能力，能快速理解电路图。

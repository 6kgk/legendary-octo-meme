# 【职教通】(Vocational Bridge) 技术架构设计提案

针对林老板提出的“极简、务实、三方获益”的核心原则，我作为软件开发负责人，对“拍搜引导”和“职业图谱”这两个核心模块进行如下技术方案设计。

## 1. 核心模块：AI 拍搜引导 (Guided AI Tutor)

该模块的核心目标是**“授人以渔”**而非“直给答案”，解决中职生基础薄弱、在校无人辅导的痛点。

### 1.1 技术实现流程
1.  **多模态采集**：学生端拍照上传。
2.  **视觉解析 (Vision LLM)**：直接调用具备 Vision 能力的大模型（如 GPT-4o 或 Gemini 1.5 Pro）进行 OCR 识别。
    *   *优势*：无需单独采购公式 OCR 服务，视觉模型能直接理解几何图形和复杂排版。
3.  **苏格拉底式 Prompt (Socratic Prompting)**：
    *   **系统预设**：严禁直接给出最终数值或答案。
    *   **启发式引导**：AI 先分析题目涉及的知识点（如：勾股定理），然后问学生：“你记得勾股定理的公式吗？”根据学生的回答（通过文字或语音）进行分步讲解。
4.  **知识降维处理**：如果学生反馈“看不懂”，Prompt 触发“降维指令”，将解释逻辑从“高一水平”退回到“初中基础”。

### 1.2 教师端友好设计
*   **思路追踪报告**：记录学生提问的分布（哪些题问得最多）和讲解过程中的卡壳点，反馈给老师，帮助老师在课堂上精准查漏补缺。

## 2. 核心模块：职业能力图谱 (Vocational Competency Map)

该模块是连接“校内学习”与“深圳实习”的数字化桥梁。

### 2.1 评价体系构建
我们将广东“3+证书”考试大纲、各类职业技能证书（如电工证、计算机证）与深圳本地岗位需求进行关联。
*   **基础素质 (Base Skills)**：语文（表达）、数学（逻辑）、英语（通用）。
*   **专业能力 (Hard Skills)**：具体证书等级、实操题正确率。
*   **职业素养 (Soft Skills)**：学习专注时长、错题复盘率。

### 2.2 数据模型
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `skill_node_id` | UUID | 知识点/技能点唯一标识 |
| `mastery_level` | Decimal | 掌握度（0-100，根据答题记录动态计算） |
| `industry_mapping` | JSON | 对应的行业/岗位（如：电子信息 -> 嘉立创质检） |
| `evidence_log` | Array | 支撑该能力的原始答题记录/证书扫描件 |

## 3. 技术栈选型建议 (Tech Stack)

*   **前端 (Mobile App)**：**Flutter**。理由：一套代码适配 Android/iOS，中职生群体手机品牌杂乱，Flutter 的兼容性表现极佳。
*   **后端 (Backend)**：**Node.js (NestJS)**。理由：适合处理高并发的 API 请求，对 AI 接口的流式响应支持良好。
*   **数据库**：**PostgreSQL + Redis**。PostgreSQL 存储核心数据和 JSON 格式的能力图谱，Redis 处理排行榜和缓存。
*   **部署环境**：**阿里云/腾讯云**。确保存储和计算节点在境内，符合数据合规要求。

## 4. 后续开发计划建议

1.  **MVP (最小可行性产品) 阶段**：实现核心的“拍搜”和“3+考题库”。
2.  **数据集成阶段**：建立题库与职业图谱的初步权重模型。
3.  **对接阶段**：接入深圳实习岗位数据流，实现简单的 AI 匹配推荐。

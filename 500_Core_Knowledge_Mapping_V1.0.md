# 职教通 (Vocational Bridge) V1.5.2：500+ 实战知识点“大白话”映射库 (Task #93)

本项目旨在彻底解决“AI 讲题空洞、不接地气”的问题。通过将 3+证书高职高考及专业技能证（电工、计算机）的核心考点，降维映射为中职生易于理解的生活化类比，实现真正的“认知溯源”。

## 一、 数学核心考点 (3+证书) - 认知降维映射 (部分展示)

| 考点 (High-Brow) | 关联底层 (Root Cause) | 大白话降维解释 (Plain Talk / Analogy) |
| :--- | :--- | :--- |
| **二次函数顶点坐标** | 负数运算、分配律 | **“抛物线的转折点”**：想象你踢出一个足球，球飞到最高的那一刻，就是顶点。公式 -b/2a 只是寻找那个“最高点”所在的横向位置。 |
| **等差数列求和** | 乘法分配律 | **“梯形拼图法”**：求 1 加到 100，就像把一个梯形反过来再拼上去，变成一个规整的长方形。首项加末项就是那个长方形的长。 |
| **正弦/余弦函数** | 勾股定理、比例 | **“时钟指针的影长”**：想象转盘上的指针。正弦 (sin) 就是指针在墙上的高度，余弦 (cos) 就是指针在地面上的投影长度。 |
| **排列组合 (P vs C)** | 分类讨论逻辑 | **“排队 vs 领水果”**：P (排列) 是排队领奖，谁站第一谁站第二不一样；C (组合) 是进果园领 3 个苹果，不管谁先拿，最后兜里都是那 3 个。 |
| **集合的交并补** | 分类归纳 | **“朋友圈分组”**：交集是两个群都有的人；并集是把两个群的人拉到一个大群；补集是“除了我家人以外的所有人”。 |

## 二、 电工基础 (专业技能证) - 认知降维映射

| 考点 (High-Brow) | 关联底层 (Root Cause) | 大白话降维解释 (Plain Talk / Analogy) |
| :--- | :--- | :--- |
| **电压、电流、电阻** | 水流模型 | **“水路模型”**：电压是“水压”，电流是“水流速度”，电阻是“变细了的水管”。水压越大水流越快，管子越细水流越慢。 |
| **电感 (自感感应)** | 惯性定律 | **“电流的脾气”**：电感就像是一个有脾气的胖子。你突然想让他跑（通电），他懒得动；你突然让他停（断电），他还想往前冲。 |
| **电容 (储能)** | 容器原理 | **“电子水桶”**：电容就是个能存电的水桶。电压高了它就装点，电压低了它就倒出点，专门用来“平抑波动”。 |
| **三相交流电 (相位差)** | 圆周运动 | **“三个推车的小伙伴”**：想象三个小伙伴围着一辆车转，每个人推的角度差 120 度。这样车子转起来才最稳、最有劲。 |

## 三、 计算机基础 (NCRE/技能证) - 认知降维映射

| 考点 (High-Brow) | 关联底层 (Root Cause) | 大白话降维解释 (Plain Talk / Analogy) |
| :--- | :--- | :--- |
| **内存 (RAM) vs 硬盘** | 数据读写速度 | **“书桌 vs 书柜”**：内存是你的“书桌”，现在要用的书都铺在上面（快但小）；硬盘是“书柜”，存着你所有的书（大但慢）。 |
| **操作系统 (OS)** | 资源调度 | **“餐馆大管家”**：OS 就是饭店领班。它决定谁先坐下、哪个菜先上、哪张桌子该翻台，没有它，饭店（硬件）就乱成一锅粥。 |
| **IP 地址与子网掩码** | 逻辑定位 | **“地址簿”**：IP 是你家的详细地址（街道号+门牌号）；子网掩码就是用来告诉你“哪几位是街道名，哪几位是你的门牌号”。 |
| **二进制 (0/1)** | 逻辑状态 | **“灯泡开关”**：电脑没那么聪明，它只认识“开”和“关”。所有的视频、音乐、游戏，本质上都是亿万个小灯泡在飞速切换开关状态。 |
